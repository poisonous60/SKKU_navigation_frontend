<Topic: SKKU 2.5D Navigation (with Road View)> [Problem] With existing navigation apps, it is difficult for users to locate specific classrooms when visiting a building for the first time. [Proposed Solution] We plan to build a 2.5D navigation system that allows users to see the interior structure of buildings and provides a walkthrough video from the starting point to the destination. We are planning to build this as a web-based service, with a separated frontend and backend architecture. The navigation will be limited to SKKU's Natural Sciences Campus. On the frontend, we will display a 2.5D map that includes building interiors. We have not yet decided on the specific tools or tech stack for 2.5D modeling and rendering. Options include creating the map from scratch using a 3D tool like Blender, or overlaying building models on top of a map service supported by Google Earth.
We will then manually assign nodes to each classroom within the buildings to construct a graph. Pathfinding will be performed using the A* algorithm on this graph. Once the user sets a starting point and a destination, the system will display the route to the classroom on the 2.5D map. Regarding the walkthrough video: we plan to record footage using an Insta360 X4 Air camera. Our current plan is to record a video for each edge in the graph, and then have the backend concatenate the videos corresponding to the edges along the computed route from start to destination. The video will be provided in 360-degree format.
